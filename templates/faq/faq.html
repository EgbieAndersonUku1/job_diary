Welcome to the job_diary wiki!

Why did I create job diary and why not use Excel to track your jobs?

The primary reason I created this web application was to keep track of the different locations for the places that I have worked as well as the different rate of pay associated with each place. Yes it is true that Excel is a great application and can be used for storing data especially in the format that I require. But the problem that led me to create this application was because over a certain period amount of time I have worked in a lot of different places which has resulted in a spreadsheet that is hundred of hundreds of rows long. This has made it exceptional difficult to query jobs based on their location, job title, hours, pay, etc.

What is job diary?

Job diary is a web application that can be run on your personal computer or laptop to keep track of the different jobs that you have worked, currently working or not yet worked. It can be used by people who work shifts, non-shifts, 9-5, etc. Once logged in the web application's home page is divided into four parts .

  • Search page
  • Entry page
  • History page
  • Active page.

The search page enables the user to search for jobs that are either active or non-active based on their location, job title, pay, hours, months, range between two months, etc.

The entry page enables you to add jobs to the database more specifically the job attributes such as the job title, the job location, the job description, the hours worked, the hourly rate, etc. There is also a shift confirmation field at the bottom. This enables you to confirm whether the shift has be confirmed or or not.

The shift confirmation field checks whether the user's shift/job has been confirmed. If confirmation is not made before the time the shift/job starts then the applications assumes that shift/job in question was cancelled by the client or by the boss. And thus automatically deletes it from the database whenever the user refreshes the page. This makes it easier on the user so he/she does not have to manually delete cancelled jobs from the application.

Example:
Assume that user has just landed a job or shift or a client job starting in a few days but the client has not yet confirmed the job. Let us further assume that job in question is on the 25/12/2016 at 10 am. Now if the current day is now equal to 25/12/2016 the day of the shift and the current time is now equal to 10 am the start of the shift and the client/boss has not confirmed then the application assumes that job has been cancelled and thus automatically deletes it from the database.

The history page shows you all jobs that you have worked in the past.

The Active page shows you all future jobs that you have yet to work.


I know this is a stupid question but how do I use the search form?

The search form contains twelve fields. The first ten fields enables the user to search the database based on the attribute of the jobs e.g. job title, location, etc.

To query by hours one must enter it as a float. For example if one worked a total of 4 hrs then the hours field must be entered as 4.0 or one worked a total of 3 hrs and 30 mins then the field must must be entered as 3.30.

To query by start time or finish time one must enter it as an hour seperated by colon followed by minutes. For example if one was query the start or finish time one would enter it as h:m e.g 9 am would be represented as 9:0

To query by date one must enter it as YYYY-MM-DD.


The bottom two field enables the user to search for job based on the ranges of two dates. This can either been as months or as dates but not a combination of both. For example you cannot use month in the first field and date in the second field or vice versa as this will result in the application returning a not found on the data even though the data might exist. The values for both fields must be the same type.
If months are used the application will return all the days worked between those two months including the starting and ending months. If date is used instead (which must be in the form of YYYY-MM-DD ) the application returns all days worked between those two dates including the starting and ending date.
